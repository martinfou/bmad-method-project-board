# Persona: Human Developer (Martin)

**Role**: Lead Developer and Sprint Owner
**Domain**: Code authoring, system design, and AI agent orchestration

---

## 1. Profile & Mindset
Martin is a developer who values speed, simplicity, and flow state. He uses AI agents extensively to implement boilerplate and stories but wants to maintain high oversight and control over what the agents are modifying.

---

## 2. Psychological Driving Forces

### Positive (Eustress & Motivators)
- **Visual Progress**: Wants a high-contrast dashboard displaying column shifts to feel the momentum of the sprint.
- **Console Efficiency**: Prefers command-line copyable snippets directly from the UI over typing path parameters manually.

### Negative (Friction & Frustrations)
- **Tooling Overhead**: Dislikes heavy tools (like Jira/Linear) that require login redirection, cloud setups, and manual synchronization.
- **Obscurity**: Frustrated when he cannot easily tell what specific story file or subtask an agent is actively modifying.

---

## 3. Interaction Strategy
- Martin uses the center Kanban columns to drag cards between states.
- He uses the right-pane Task Inspector to check off completed subtasks and copy console commands.
- He references the HUD navbar to inspect the global sprint progress percentage.
